1. BACKUP KONFIGURASI DI MIKROTIK

Ilustrasi permission read pada MikroTik RouterOS

Dari sumber MikroTik Documentation (2023), backup konfigurasi adalah proses menyimpan seluruh pengaturan router dalam format biner yang dapat dengan mudah dipulihkan. Format ini menyimpan semua data konfigurasi secara lengkap dan terenkripsi.

Perintah dasar untuk melakukan backup melalui CLI adalah:

/system backup save name=nama_backup
# Membuat file backup dengan nama tertentu

Selalu lindungi file backup dengan password untuk mencegah unauthorized access terhadap konfigurasi router Anda.

Jika ingin menambahkan password untuk melindungi file backup, gunakan:

/system backup save name=nama_backup password=kata_sandi
# Backup dengan proteksi password

Dari sumber OperaVPS (2025), pada RouterOS versi lama (sebelum v6.43), jika password tidak diberikan, file backup akan dienkripsi dengan password user yang sedang aktif.

Untuk membuat backup tanpa enkripsi, gunakan parameter dont-encrypt=yes:

/system backup save name=nama_backup dont-encrypt=yes
# Backup tanpa enkripsi (tidak direkomendasikan)

Dari sumber ShellHacks (2023), file backup yang dihasilkan akan tersimpan di penyimpanan router dan dapat dilihat menggunakan perintah:

/file print
# Melihat daftar file di penyimpanan router

Untuk mengunduh file backup, Anda dapat menggunakan protokol FTP atau SFTP, atau dengan melakukan drag-and-drop melalui WinBox.

Kesimpulan: Backup konfigurasi MikroTik dalam format biner (.backup) merupakan cara cepat dan efisien untuk menyimpan seluruh konfigurasi perangkat, dengan opsi tambahan untuk mengamankannya menggunakan password. Praktik terbaik adalah melakukan backup secara berkala dan menyimpannya di lokasi yang aman.

2. RESTORE KONFIGURASI DI MIKROTIK

Ilustrasi permission read pada MikroTik RouterOS

Dari sumber MikroTik Documentation (2023), proses restore adalah tindakan mengembalikan konfigurasi router dari file backup yang sebelumnya telah dibuat. Perlu diperhatikan bahwa restore dari file backup sebaiknya dilakukan pada perangkat yang sama dan dengan versi RouterOS yang sama.

Untuk melakukan restore melalui CLI, gunakan perintah:

/system backup load name=nama_backup
# Mengembalikan konfigurasi dari file backup

Jika file backup dilindungi password:

/system backup load name=nama_backup password=kata_sandi
# Restore dengan password

Dari sumber OperaVPS (2025), setelah menjalankan perintah restore, router akan meminta konfirmasi untuk me-restart perangkat, karena restore memerlukan reboot untuk menerapkan konfigurasi yang dipulihkan.

# Prompt konfirmasi yang akan muncul
Restore and reboot? [y/N]:
# Ketik 'y' untuk melanjutkan

Dari sumber Tech@Layer-x (2024), jika ingin melakukan restore pada perangkat yang berbeda, disarankan untuk menggunakan metode export/import daripada backup/restore, karena file backup berisi informasi yang spesifik untuk perangkat tertentu seperti MAC address.

Kesimpulan: Restore konfigurasi dari file backup adalah metode cepat untuk mengembalikan konfigurasi perangkat ke kondisi sebelumnya, namun sebaiknya dilakukan pada perangkat yang sama untuk menghindari masalah kompatibilitas. Proses ini akan merestart router secara otomatis untuk menerapkan konfigurasi yang dipulihkan.

3. EXPORT/IMPORT KONFIGURASI DI MIKROTIK

Ilustrasi permission read pada MikroTik RouterOS

Dari sumber MikroTik Documentation (2023), export konfigurasi menghasilkan file teks (.rsc) yang berisi serangkaian perintah CLI yang dapat digunakan untuk menerapkan konfigurasi. Berbeda dengan backup, export dapat dilakukan sebagian atau seluruhnya dan menghasilkan file yang dapat dibaca dan diedit.

Untuk mengexport seluruh konfigurasi:

/export file=nama_file
# Export semua konfigurasi ke file .rsc

Untuk mengexport konfigurasi kompak (hanya yang berbeda dari default):

/export compact file=nama_file
# Export konfigurasi yang tidak default saja

Export konfigurasi dalam format .rsc memungkinkan Anda untuk mengedit konfigurasi secara manual sebelum mengimportnya kembali, sangat berguna untuk modifikasi dan transfer antar perangkat berbeda.

Untuk mengexport konfigurasi tertentu saja, misalnya hanya firewall:

/ip firewall export file=firewall_config
# Export konfigurasi firewall saja

Dari sumber Tutorial Jaringan (2017), jika ingin menyembunyikan informasi sensitif seperti password:

/export file=nama_file hide-sensitive
# Export tanpa informasi sensitif

Dari sumber MikroTik Documentation (2023), untuk melakukan import konfigurasi dari file .rsc:

/import file=nama_file.rsc
# Import konfigurasi dari file .rsc

Untuk import dengan opsi verbose (menampilkan detail proses import):

/import file=nama_file.rsc verbose=yes
# Import dengan tampilan detail proses

Dari sumber Indonesia Go Network (2022), pastikan untuk memeriksa file .rsc sebelum mengimportnya, terutama jika berasal dari sumber eksternal, untuk menghindari konfigurasi yang berpotensi berbahaya.

Ada beberapa parameter tambahan yang dapat digunakan saat import:

/import file=nama_file.rsc skip-duplicate-entries=yes
# Melewati entri yang sudah ada

Untuk mensimulasikan import tanpa menerapkan perubahan (untuk mendeteksi kesalahan):

/import file=nama_file.rsc verbose=yes dry-run=yes
# Simulasi import tanpa menerapkan perubahan
Kesimpulan: Export/import konfigurasi dengan format .rsc memberikan fleksibilitas lebih tinggi dibandingkan backup/restore, memungkinkan transfer konfigurasi antar perangkat berbeda, dan memberi kesempatan untuk mengedit konfigurasi sebelum menerapkannya. Metode ini juga memungkinkan export/import sebagian konfigurasi saja.
